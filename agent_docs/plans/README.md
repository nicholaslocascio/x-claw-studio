# Planning Docs

These plans break the next strategy layer into small, separable systems.

- [`01-format-classification.md`](./01-format-classification.md): detect repeatable post shapes such as quote-tweet-with-video, thread, reaction image, or screenshot post.
- [`02-format-radar.md`](./02-format-radar.md): rank current winning formats separately from topic heat.
- [`03-relative-performance.md`](./03-relative-performance.md): normalize engagement so small and large accounts can be compared fairly.
- [`04-source-response-graph.md`](./04-source-response-graph.md): preserve what a post is responding to, quoting, or reframing.
- [`05-outcome-memory.md`](./05-outcome-memory.md): learn from published drafts and actual results over time.

Read these after the main agent docs, not instead of them. They are forward-looking implementation plans, not current-state architecture docs.
